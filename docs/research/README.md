# Research

This directory holds the literature review and research groundwork for the project. Phase 0 creates the skeleton files below; substantive content is filled in during **Phase 1 — Research Deep Dive** (see `TASK_BOARD.md`).

## Contents

- [related-work.md](related-work.md) — literature review of prior work on LLM/RAG security.
- [owasp-llm-top10-mapping.md](owasp-llm-top10-mapping.md) — mapping of OWASP Top 10 for LLM Applications to this project's threats.
- [llmsvs-checklist.md](llmsvs-checklist.md) — adapted security verification checklist for the gateway.
- [tool-comparison.md](tool-comparison.md) — comparison of guardrail libraries, RAG frameworks, and vector stores under consideration.
- [dataset-review.md](dataset-review.md) — review of existing public red-team / prompt injection datasets relevant to synthetic test data design.

## Ground rules

- All citations must be real and verifiable — no fabricated papers, authors, or links (`AGENT_RULES.md` rule 2).
- No benchmark numbers are copied into these docs as if they were this project's own results — clearly attribute any numbers to their original source.
- Keep entries dated so it's clear when a source was reviewed.
