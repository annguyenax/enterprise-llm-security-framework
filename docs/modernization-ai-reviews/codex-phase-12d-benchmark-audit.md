## Critical Issues

### 1. Ground truth is gated by current guard behavior
- exact file/artifact: `scripts/validate_v2_benchmark.py::check_guard_cross_reference`, `main`; generated label rationales
- evidence: Real guards are imported and mismatches enter `all_errors`; executable probe returned non-zero solely for disagreement with the current Input Guard.
- impact: Circular validation can reject independent ground truth and encourages labels/templates to match current rules, invalidating generalization claims.
- minimal fix: Remove guard cross-checking from validation success. If retained, emit a clearly named non-gating diagnostic report only.
- required regression test: A schema-valid benchmark that disagrees with current guards must still pass integrity validation while the optional diagnostic reports the mismatch.
- blocking: yes

### 2. Holdout is materially contaminated by shared templates
- exact file/artifact: `scripts/build_v2_benchmark.py::FAMILY_TABLE`, `build_all`, family builders; validation and holdout JSONL artifacts
- evidence: Same builders generate all splits sequentially; 34/60 holdout queries score at least 0.9 similarity and 17/23 families share normalized templates. One validation prompt closely restates v1.
- impact: Holdout results cannot support the intended unseen-generalization claim.
- minimal fix: Author independent validation/holdout formulations using separate templates and review provenance; remove v1-derived validation content.
- required regression test: Template-fingerprint and similarity checks across splits and against v1, with reviewed thresholds and explicit exemptions.
- blocking: yes

## Major Issues

### 1. Validator does not enforce its complete schema/integrity contract
- exact file/artifact: `scripts/validate_v2_benchmark.py::check_schemas`, `check_category_coverage`, `main`
- evidence: Invalid Decision and unknown label fields passed; globally missing family passed; dangling/mismatched references crashed.
- impact: Malformed or incomplete frozen artifacts may be accepted, while some invalid inputs expose stack traces.
- minimal fix: Validate exact label fields, enums, per-record split/language consistency, required family registry, and one-to-one mappings before dependent checks.
- required regression test: CLI-level non-zero, deterministic-error tests for every required negative probe.
- blocking: yes

### 2. Label isolation and evaluation scope are incomplete
- exact file/artifact: corpus `expected_ingestion_status`; case/label schemas
- evidence: Expected ingestion outcome is stored in corpus; no `evaluation_scope` exists.
- impact: Phase 12E must infer execution mode from family names and prose, risking accidental HTTP claims for component/residual scenarios.
- minimal fix: Move expected ingestion outcome into labels and add a validated scope such as `end_to_end`, `component`, `availability_fault`, or `residual_risk_only`.
- required regression test: No `expected_*` fields in corpus/cases and complete scope coverage for every case.
- blocking: yes

### 3. Claimed balance is weak
- exact file/artifact: `FAMILY_TABLE`, benchmark methodology
- evidence: 36 benign versus 74 malicious, 6 mixed, and 4 neutral; holdout is 18 benign versus 38 malicious.
- impact: The corpus is approximately 30% benign, weakening “approximately balanced” wording and precision of FPR comparisons.
- minimal fix: Rebalance or amend the governing claim before freezing.
- required regression test: Explicit reviewed category bounds.
- blocking: yes
